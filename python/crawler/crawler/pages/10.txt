<h2>Tomcat</h2>Почему именно Tomcat а не <a href="https://glassfish.java.net/">Glassfish V3</a>? Для меня важно использование именно Tomcat т.к. он компактен, быстро работает и легко объединяется в кластер под управлением Apache Httpd. V3 <a href="http://wiki.glassfish.java.net/Wiki.jsp?page=GlassFishV3Clustering">пока</a> не умеет работать в кластере.<h3>Установка</h3><ol>
<li>Установите <a href="http://java.sun.com/javase/downloads/widget/jdk6.jsp">JDK 6</a>.</li>
<li>Настройте переменную окружения JAVA_HOME</li>
<li>Скачайте архив <a href="http://www.sai.msu.su/apache/tomcat/tomcat-6/v6.0.26/bin/apache-tomcat-6.0.26.zip">apache-tomcat-6.0.26.zip</a> и распакуйте в текущую папку. Будет создана папка apache-tomcat-6.0.26.</li>
</ol><strong>Для запуска:</strong><blockquote><code>> bin/startup.(bat|sh) - откроет новое окно или&lt;br/>> bin/catalina.(bat|sh) run - запустит в новом окне.</code></blockquote>Для увеличения памяти и возможности отладки перед запуском catalina.bat можно указать переменную окружения JAVA_OPTS:<blockquote><code>set JAVA_OPTS=-Xmx512m -XX:MaxPermSize=256m -Xdebug -Xrunjdwp:transport=dt_socket,address=1025,server=y,suspend=n&lt;br/>export JAVA_OPTS=-Xmx512m -XX:MaxPermSize=256m -Xdebug -Xrunjdwp:transport=dt_socket,address=1025,server=y,suspend=n</code></blockquote><br/>
<h2>Unified EL 2.2</h2>Язык выражений нужен как в JSP при использовании библиотек тэгов, так и в facelets. Основные улучшения в новой версии (см. <a href="http://seamframework.org/Documentation/EL21">список нововведений</a>) — возможность вызывать методы, в т.ч. с параметрами.<blockquote><code><font color="black">${application.getRealPath(<font color="#A31515">'/data'</font>)}</font></code></blockquote><h3>Установка</h3><ol>
<li>Удалите lib/el-api.jar из Tomcat.</li>
<li>Скопируйте реализацию <a href="http://download.java.net/maven/glassfish/javax/el/el-api/2.2.0-SNAPSHOT/el-api-2.2.0-SNAPSHOT.jar">el-api-2.2.jar</a> и <a href="http://download.java.net/maven/glassfish/org/glassfish/web/el-impl/2.2.0-SNAPSHOT/el-impl-2.2.0-SNAPSHOT.jar">el-impl-2.2.jar</a> в папку lib в Tomcat. Это jar-ы из Glassfish V3.</li>
<li>Добавьте в WEB-INF/web.xml в вашем приложении следующие строки: <blockquote><code><font color="black"><font color="#0000ff">&lt;</font><font color="#800000">context-param</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">param-name</font><font color="#0000ff">></font>com.sun.faces.expressionFactory<font color="#0000ff">&lt;/</font><font color="#800000">param-name</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">param-value</font><font color="#0000ff">></font>com.sun.el.ExpressionFactoryImpl<font color="#0000ff">&lt;/</font><font color="#800000">param-value</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">context-param</font><font color="#0000ff">></font></font></code></blockquote></li>
</ol>К сожалению после обновления языка выражений Jasper (реализация компилятора JSP в Tomcat) не понимает новые выражения и они не могут быть использованы в JSP. Если вы хотите, чтобы Tomcat везде использовал новый язык выражений (а не свою реализацию org.apache.el) нужно перекомпилировать Tomcat. Подробности <a href="http://huksley.sdot.ru/141/tomcat-and-unified-e22.html">здесь</a> (на англ. языке). Также пункт 3 будет не нужен.<br/>
<h2>Mojarra</h2>Первая реализация JSF 2. Стабильно и быстро работает. Помимо реализации JSF нам также понадобится установить JSTL 1.2.<h3>Установка</h3><ol>
<li>Скачайте <a href="https://javaserverfaces.dev.java.net/files/documents/1866/146040/mojarra-2.0.2-FCS-binary.zip">mojarra-2.0.2-FCS-binary.zip</a></li>
<li>Скопируйте из архива из папки mojarra-2.0.2-FCS/lib jsf-api.jar и jsf-impl.jar в папку WEB-INF/lib вашего приложения.</li>
<li>Скопируйте <a href="https://maven-repository.dev.java.net/repository/jstl/jars/jstl-1.2.jar">jstl 1.2</a> в папку WEB-INF/lib вашего приложения.</li>
<li>Добавьте в WEB-INF/web.xml в вашем приложении следующие строки:<blockquote><code><font color="black"><font color="#0000ff">&lt;</font><font color="#800000">servlet</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">servlet-name</font><font color="#0000ff">></font>FacesServlet<font color="#0000ff">&lt;/</font><font color="#800000">servlet-name</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">servlet-class</font><font color="#0000ff">></font>javax.faces.webapp.FacesServlet<font color="#0000ff">&lt;/</font><font color="#800000">servlet-class</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">load-on-startup</font><font color="#0000ff">></font>1<font color="#0000ff">&lt;/</font><font color="#800000">load-on-startup</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">servlet</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">servlet-mapping</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">servlet-name</font><font color="#0000ff">></font>FacesServlet<font color="#0000ff">&lt;/</font><font color="#800000">servlet-name</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">url-pattern</font><font color="#0000ff">></font>*.xhtml<font color="#0000ff">&lt;/</font><font color="#800000">url-pattern</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">servlet-mapping</font><font color="#0000ff">></font></font></code></blockquote></li>

</ol><br/>
<h2>Weld</h2>Weld реализует спецификацию <a href="http://jcp.org/en/jsr/detail?id=299">JSR 299</a> Contexts and Dependency Injection.Вкратце Weld позволяет устанавливать значения полей автоматически с помощью аннотаций.<h3>Установка</h3><ol>
<li>Скачайте <a href="https://sourceforge.net/projects/jboss/files/Weld/1.0.1.Final/weld-1.0.1-Final.zip/download">weld-1.0.1-Final.zip</a></li>
<li>Создайте файл WEB-INF/beans.xml со следующим содержимым:<blockquote><code><font color="black"><font color="#0000ff">&lt;?</font><font color="#800000">xml</font> <font color="#ff0000">version</font><font color="#0000ff">="1.0"</font> <font color="#ff0000">encoding</font><font color="#0000ff">="UTF-8"</font>?<font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">beans</font><font color="#0000ff">/></font></font></code></blockquote>Это нужно чтобы Weld сканировал веб-приложение на предмет Java-Beans которые будут участвовать в CDI.</li>
<li>Скопируйте weld-1.0.1-Final/artifacts/weld/weld-tomcat-support.jar из архива в lib папку в Tomcat.</li>
<li>Cоздайте в вашем приложении файл META-INF/context.xml со следующим содержимым:<blockquote><code><font color="black"><font color="#0000ff">&lt;?</font><font color="#800000">xml</font> <font color="#ff0000">version</font><font color="#0000ff">="1.0"</font> <font color="#ff0000">encoding</font><font color="#0000ff">="UTF-8"</font>?<font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">Context</font><font color="#0000ff">></font>&lt;br/><font>&lt;!-- Declare Weld Bean Manager --></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">Resource</font> <font color="#ff0000">name</font><font color="#0000ff">="BeanManager"</font>&lt;br/>    <font color="#ff0000">auth</font><font color="#0000ff">="Container"</font>&lt;br/>    <font color="#ff0000">type</font><font color="#0000ff">="javax.enterprise.inject.spi.BeanManager"</font>&lt;br/>    <font color="#ff0000">factory</font><font color="#0000ff">="org.jboss.weld.resources.ManagerObjectFactory"</font><font color="#0000ff">/></font>&lt;br/><font>&lt;!-- For Weld to support Servlet Injection --></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">Listener</font> <font color="#ff0000">className</font><font color="#0000ff">="org.jboss.weld.environment.tomcat.WeldLifecycleListener"</font><font color="#0000ff">/></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">Context</font><font color="#0000ff">></font></font></code></blockquote></li>

<li>Добавьте в WEB-INF/web.xml следующие строки:<blockquote><code><font color="black"><font>&lt;!-- Weld listener --></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">listener</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">listener-class</font><font color="#0000ff">></font>org.jboss.weld.environment.servlet.Listener<font color="#0000ff">&lt;/</font><font color="#800000">listener-class</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">listener</font><font color="#0000ff">></font>&lt;br/><font>&lt;!-- Reference to Weld Bean Manager --></font>&lt;br/><font color="#0000ff">&lt;</font><font color="#800000">resource-env-ref</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">description</font><font color="#0000ff">></font>Object factory for the CDI Bean Manager<font color="#0000ff">&lt;/</font><font color="#800000">description</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">resource-env-ref-name</font><font color="#0000ff">></font>BeanManager<font color="#0000ff">&lt;/</font><font color="#800000">resource-env-ref-name</font><font color="#0000ff">></font>&lt;br/>    <font color="#0000ff">&lt;</font><font color="#800000">resource-env-ref-type</font><font color="#0000ff">></font>javax.enterprise.inject.spi.BeanManager<font color="#0000ff">&lt;/</font><font color="#800000">resource-env-ref-type</font><font color="#0000ff">></font>&lt;br/><font color="#0000ff">&lt;/</font><font color="#800000">resource-env-ref</font><font color="#0000ff">></font></font></code></blockquote></li>

<h2>Hibernate Validator</h2>JSR 303 Bean Validation дает простой способ применения правил валидации как для JSF так и для JPA. Данные правила определяются через <a href="http://docs.jboss.org/hibernate/stable/validator/reference/en/html/validator-usingvalidator.html#table-builtin-constraints">аннотации</a> к полям в Java Beans:<blockquote><code><font color="black">@NotNull@Size(min = 1)&lt;br/><font color="#0000ff">private</font> <font color="#2B91AF">String</font> username;</font></code></blockquote><h3>Установка</h3><ol>
<li>Скачайте <a href="http://sourceforge.net/projects/hibernate/files/hibernate-validator/4.0.2.GA/hibernate-validator-4.0.2.GA-dist.zip/download">hibernate-validator-4.0.2.GA-dist.zip</a></li>
<li>Скопируйте hibernate-validator-4.0.2.GA/hibernate-validator-4.0.2.GA.jar из архива в папку WEB-INF/lib вашего приложения.</li>
<li>Скопируйте из папки в архиве hibernate-validator-4.0.2.GA/lib дополнительные библиотеки slf4j-api-1.5.6.jar, validation-api-1.0.0.GA.jar в папку WEB-INF/lib вашего приложения.</li>
</ol><br/>
<h2>EclipseLink</h2>JPA позволяет сохранять в БД Java Beans без использования SQL. Нужные вызовы и если нужно схема БД создается автоматически.
